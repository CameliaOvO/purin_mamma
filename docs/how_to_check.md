# 웹페이지 확인 방법

이 프로젝트는 별도 빌드 과정이 없는 정적 웹페이지입니다. 아래 방법 중 하나로 확인할 수 있습니다.

## 1. 가장 빠른 로컬 확인

터미널에서 프로젝트 폴더로 이동한 뒤 아래 명령어를 실행합니다.

```bash
python3 -m http.server 8000
```

브라우저에서 아래 주소를 엽니다.

```text
http://127.0.0.1:8000/
```

정상이라면 다음 내용을 볼 수 있습니다.

- `이번 달 이유식 달력`이라는 큰 제목
- 푸린이의 오늘 기준 생후 일차 카드와 이유식 시작일 입력칸
- 이전 달 / 이번 달 / 다음 달 버튼이 있는 월간 이유식 달력
- 날짜를 눌렀을 때 바뀌는 선택 날짜 상세 패널
- `웹으로 만들면 접근성과 유지보수가 쉬워요` 설명 섹션
- `식재료 변경 시` 흐름 설명
- 하단 의료 주의 문구

확인을 마쳤으면 터미널에서 `Ctrl + C`를 눌러 서버를 종료합니다.

## 2. 클라우드 개발 환경에서 미리보기

Codespaces, Gitpod, Codex 같은 클라우드 개발 환경에서는 `http://127.0.0.1:8000/` 주소가 내 컴퓨터가 아니라 **클라우드 컨테이너 내부**를 가리키기 때문에 브라우저에서 바로 보이지 않을 수 있습니다.

이 경우 아래 순서로 확인합니다.

1. 터미널에서 서버를 실행합니다.

   ```bash
   python3 -m http.server 8000
   ```

2. 개발 환경의 **Ports**, **Forwarded Ports**, **Preview**, **Open in Browser** 메뉴에서 `8000` 포트를 엽니다.
3. 표시되는 forwarded URL 또는 preview URL로 접속합니다.
4. preview 창이 비어 있으면 새 브라우저 탭에서 forwarded URL을 직접 열어봅니다.

서버 자체가 정상인지 터미널에서 먼저 확인하려면 아래 명령어를 사용할 수 있습니다.

```bash
curl -I http://127.0.0.1:8000/
```

`200 OK`가 나오면 웹페이지 서버는 정상이고, 문제는 대부분 클라우드 미리보기/포트 전달 설정 쪽입니다.

## 3. GitHub Pages로 공개 확인

GitHub 저장소에서 아래 순서대로 설정합니다.

1. 저장소 화면에서 **Settings**를 엽니다.
2. 왼쪽 메뉴에서 **Pages**를 선택합니다.
3. **Build and deployment**의 Source를 `Deploy from a branch`로 선택합니다.
4. Branch를 현재 작업 브랜치 또는 기본 브랜치로 선택합니다.
5. 폴더는 `/ (root)`를 선택합니다.
6. **Save**를 누릅니다.
7. 잠시 뒤 표시되는 `https://<username>.github.io/<repository>/` 주소를 엽니다.

## 4. 휴대폰에서 앱처럼 확인

GitHub Pages 주소가 생긴 뒤 휴대폰 브라우저에서 접속합니다.

- iPhone Safari: 공유 버튼 → **홈 화면에 추가**
- Android Chrome: 메뉴 → **홈 화면에 추가** 또는 **앱 설치**

이렇게 하면 아직 네이티브 앱은 아니지만, 홈 화면 아이콘으로 빠르게 열 수 있습니다.

## 5. 문제가 생겼을 때

### 브라우저에서 페이지가 안 열릴 때

- 로컬 서버 명령어를 실행한 터미널이 계속 켜져 있는지 확인합니다.
- 로컬 PC에서는 주소가 `http://127.0.0.1:8000/`인지 확인합니다.
- 클라우드 개발 환경에서는 `8000` 포트가 외부로 전달되었는지 확인합니다.
- 8000 포트가 이미 사용 중이면 다음처럼 다른 포트를 씁니다.

```bash
python3 -m http.server 8080
```

그다음 로컬 PC에서는 `http://127.0.0.1:8080/`, 클라우드 개발 환경에서는 `8080` 포트의 forwarded URL을 엽니다.

### GitHub Pages 주소가 404일 때

- Pages 설정에서 폴더가 `/ (root)`인지 확인합니다.
- `index.html` 파일이 저장소 루트에 있는지 확인합니다.
- 배포 반영까지 몇 분 정도 기다린 뒤 새로고침합니다.
