# 아기 이유식 식단표 달력 요구사항

## 1. 제품 목표

`purin_mamma`는 보호자가 아기의 이유식 식단을 달력 형태로 계획하고, 실제 섭취 결과와 알레르기 반응을 기록하며, 중간에 식재료가 변경되었을 때 이후 식단을 일관되게 재계산할 수 있도록 돕는 서비스다.

> 의료 주의: 이 서비스는 식단 계획과 기록을 보조하는 도구이며, 진단·치료·응급 대응을 대체하지 않는다. 알레르기 고위험군, 기존 알레르기, 심한 습진, 호흡기 증상, 반복 구토, 전신 두드러기 등 우려 상황은 반드시 소아청소년과 또는 알레르기 전문의 상담을 우선한다.

## 2. 핵심 사용자 시나리오

1. 보호자는 아기의 생년월일, 이유식 시작일, 식사 단계, 수유/분유 여부, 피해야 할 식재료, 선호 식재료를 입력한다.
2. 시스템은 월간/주간 달력에 끼니별 이유식 메뉴와 식재료 구성을 제안한다.
3. 알레르기 테스트가 필요한 식재료는 “신규 도입”, “관찰 중”, “통과”, “보류”, “반응 있음” 상태로 관리한다.
4. 보호자는 실제 섭취량, 반응, 배변, 컨디션, 사진, 메모를 기록한다.
5. 중간에 식재료를 교체하거나 제외하면 시스템은 해당 날짜 이후의 메뉴, 장보기 목록, 알레르기 테스트 일정, 영양 다양성 점검을 함께 갱신한다.

## 3. 알레르기 도입 원칙

최초 구현은 아래 보수적 원칙을 기본값으로 사용한다.

- 대부분의 아기는 발달 준비가 되었을 때 보통 생후 약 6개월 무렵부터 고형식을 시작하며, 달걀·땅콩·유제품·참깨 같은 알레르기 유발 가능 식품을 지연 도입하는 것이 알레르기 예방에 도움이 된다는 근거는 없다.
- 땅콩의 경우 심한 습진 또는 달걀 알레르기가 있는 고위험 영아는 의학적 평가 후 생후 4~6개월부터 연령에 맞는 형태로 도입하는 지침을 별도 안내한다.
- 경증 또는 중등도 습진이 있는 아기는 보통 생후 약 6개월 무렵 땅콩을 도입할 수 있으나, 보호자가 불안하거나 병력이 복잡하면 의료진 상담을 권장한다.
- 온전한 견과류, 큰 덩어리, 끈적한 땅콩버터 원액 등 질식 위험이 있는 형태는 금지하고, 묽게 풀거나 이유식에 소량 섞는 등 연령에 맞는 형태만 허용한다.
- 신규 식재료는 가능하면 오전 또는 낮 시간에 도입하고, 같은 날 여러 신규 고위험 식재료를 겹치지 않게 배치한다.
- 한 번 통과한 주요 알레르기 식재료는 식단에서 완전히 사라지지 않도록 주기적인 유지 노출 일정을 제안한다.

## 4. 식재료 상태 모델

| 상태 | 설명 | 식단 생성 영향 |
| --- | --- | --- |
| `candidate` | 도입 후보 | 앞으로 테스트 슬롯에 배정 가능 |
| `scheduled` | 특정 날짜에 신규 도입 예정 | 해당 날짜 이후 관찰 기간 생성 |
| `observing` | 도입 후 반응 관찰 중 | 다른 신규 고위험 식재료와 겹치지 않음 |
| `passed` | 문제 없이 섭취 | 반복 노출 및 일반 메뉴에 사용 가능 |
| `paused` | 보호자가 임시 보류 | 이후 메뉴에서 제외하고 대체 후보 필요 |
| `reacted` | 의심 반응 기록 | 이후 메뉴에서 제외하고 의료 상담 안내 |
| `excluded` | 장기 제외 | 메뉴·장보기 목록·추천에서 제외 |

## 5. 달력 생성 규칙

### 5.1 입력값

- 아기 정보: 생년월일, 교정월령 여부, 이유식 시작일, 현재 단계
- 식사 설정: 하루 식사 횟수, 질감, 1회 목표량, 선호 조리 방식
- 제한 조건: 알레르기 병력, 습진 정도, 가족 선호, 종교/문화적 제한, 사용 금지 식재료
- 운영 조건: 장보기 주기, 조리 가능 요일, 냉동 보관 허용 기간

### 5.2 생성 순서

1. 이미 통과한 안전 식재료로 기본 베이스 메뉴를 만든다.
2. 신규 식재료 테스트 슬롯을 주 1~3회 범위에서 배치한다.
3. 주요 알레르기 식재료는 위험도와 보호자 설정에 따라 의료 상담 필요 여부를 표시한다.
4. 같은 기간에 곡류·채소·단백질·지방·과일의 다양성이 지나치게 편중되지 않도록 조정한다.
5. 조리·보관·장보기 효율을 위해 인접 날짜에는 호환 가능한 식재료를 재사용하되, 신규 테스트 식재료는 추적 가능하게 표시한다.

## 6. 중간 식재료 변경 전파 규칙

보호자가 특정 날짜의 식재료를 바꾸면 시스템은 변경 범위를 명확히 물어본 뒤 이후 계획을 갱신한다.

### 6.1 변경 범위 옵션

- `single_meal`: 해당 끼니만 변경
- `same_day`: 해당 날짜 전체 변경
- `future_from_date`: 해당 날짜 이후 모든 계획에 반영
- `test_sequence_only`: 알레르기 테스트 순서만 재계산
- `shopping_only`: 장보기 목록만 갱신

### 6.2 전파 알고리즘

1. 변경된 식재료의 상태를 확인한다.
2. 제거된 식재료가 `scheduled` 또는 `observing`이면 관찰 일정을 취소하거나 뒤로 미룬다.
3. 추가된 식재료가 신규 식재료이면 해당 날짜를 테스트 시작일로 등록하고 관찰 기간을 만든다.
4. 추가된 식재료가 `reacted`, `paused`, `excluded` 상태이면 경고를 표시하고 자동 반영하지 않는다.
5. 이후 메뉴에서 제거 식재료를 사용하는 모든 끼니를 찾는다.
6. 영양군, 질감, 조리법, 이미 통과한 식재료 여부를 기준으로 대체 후보를 고른다.
7. 대체 결과가 알레르기 테스트 충돌을 만들면 가장 가까운 빈 테스트 슬롯으로 이동한다.
8. 갱신된 식단, 테스트 일정, 장보기 목록, 변경 이력을 한 번에 저장한다.

## 7. 변경 이력과 되돌리기

모든 자동 변경은 보호자가 검토할 수 있도록 diff 형태로 보여준다.

- 변경 전/후 메뉴
- 변경 사유
- 영향을 받은 날짜 수
- 새로 추가되거나 취소된 알레르기 테스트
- 장보기 목록 증감
- “적용”, “부분 적용”, “취소”, “되돌리기” 액션

## 9. MVP 범위

- 월간/주간 식단 달력
- 식재료별 알레르기 테스트 상태 관리
- 신규 식재료 도입 일정 자동 배치
- 식재료 변경 시 이후 식단 재계산
- 변경 diff 확인 및 되돌리기
- 장보기 목록 자동 갱신
- 의료 상담 필요 플래그와 응급 경고 문구

## 10. 후속 검토 항목

- 한국 소아청소년과 또는 알레르기 학회 지침을 별도 출처로 추가 검토
- 월령별 질감·입자 크기·질식 위험 식재료 데이터베이스 구축
- 보호자별 위험 허용도에 따른 테스트 간격 설정
- 소아과 상담 기록 또는 의사 권고사항을 우선 규칙으로 고정하는 기능
