Web first

웹으로 만들면 접근성과 유지보수가 쉬워요

초기에는 로그인, 서버, 앱스토어 심사 없이 정적 페이지로 식단표와 규칙을 검증하는 것이 좋습니다. 실제로 자주 쓰게 되면 저장 기능, 가족 공유, PWA 알림을 단계적으로 추가할 수 있습니다.

웹 MVP

빠른 배포, 쉬운 공유, GitHub Pages 연결에 유리합니다.

네이티브 앱

푸시 알림과 기기 기능은 강하지만 초기 개발·배포 부담이 큽니다.

식재료 변경 시

  1. 변경 범위를 선택합니다.
  2. 알레르기 테스트 충돌을 확인합니다.
  3. 이후 메뉴와 장보기 목록을 재계산합니다.
  4. 변경 전/후 diff를 보고 적용합니다.

Safe by default

중간에 바꿔도 이후 계획이 따라오게

예를 들어 닭고기를 소고기로 바꾸면, 이후 닭고기가 들어간 끼니를 찾고 이미 통과한 대체 식재료 중심으로 다시 배치하도록 설계합니다.

의료 주의

이 페이지는 식단 계획을 돕는 도구입니다. 알레르기 고위험군이거나 의심 반응이 있으면 소아청소년과 또는 알레르기 전문의와 상담하세요.
